## Running the E2E Script

```bash
# Runs the full learner journey: register → login → search → add to basket
node e2e/learner-journey.spec.js
```

**What it tests:**
1. Navigates to the app (URL read from `.env`)
2. Registers a new user with a unique timestamped email
3. Logs in with the newly registered credentials
4. Searches for a product by keyword
5. Opens the product detail page
6. Adds the product to the basket
7. **Asserts the basket count increments** — this assertion fails if the enrollment flow is broken

On failure, screenshots are saved to `./screenshots/`.

**Expected exit codes:**
- `0` — All assertions passed
- `1` — One or more assertions failed (check stdout for details)

## Running the k6 Load Script

```bash
# Basic run
k6 run --env BASE_URL=https://with-bugs.practicesoftwaretesting.com load/k6/search-load.js

# With Prometheus remote-write (requires xk6-prometheus build)
k6 run --out experimental-prometheus-rw load/k6/search-load.js
```

**Load profile:**
- Ramp up: 0 → 20 VUs over 10 s
- Hold: 20 VUs for 30 s
- Ramp down: 20 → 0 over 10 s

**Thresholds:**
- `p(95) < 2 000 ms` — 95th percentile response time
- `error rate < 1%` — HTTP errors + failed checks

## Running the JMeter Plan

```bash
# CLI mode (headless)
jmeter -n -t load/jmeter/search-api-plan.jmx \
       -l load/jmeter/results.jtl \
       -e -o load/jmeter/html-report/
```

Or open `load/jmeter/search-api-plan.jmx` in the JMeter GUI to view/run interactively.

**Plan covers:**
- 10 concurrent users, 60-second run
- 3 assertions: HTTP 200 status, response time < 2 s, JSON body contains `data`
- Results summary screenshot committed at `load/jmeter/results-summary.png`

## Prometheus Metrics

When running k6 with the Prometheus output, the following metrics are available in the Prometheus UI (`http://localhost:9090`):

| Metric                  | Description                         |
|-------------------------|-------------------------------------|
| `k6_http_req_duration`  | Request duration histogram          |
| `k6_custom_error_rate`  | Custom error rate gauge             |
| `k6_search_response_time` | Search-specific response trend    |

Screenshot of the `k6_http_req_duration` metric visible in Prometheus UI is at `load/k6/prometheus-screenshot.png`.

## Notes on Load Testing Targets

All load tests target the **practice app** URL above.  
In a production project, load tests run against a **dedicated staging environment only** — never production. The `BASE_URL` env var enforces this separation: change the URL in `.env` without touching any script.
